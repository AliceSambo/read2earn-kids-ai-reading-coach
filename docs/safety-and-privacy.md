# Safety, privacy, and accessibility boundary

This prototype is privacy-minimising by design. It has no registration, analytics, advertising, payment, social graph, messaging, or cloud database. Child and companion nicknames are stored only in the current browser. Story content and learning data are local sample data.

The prototype asks for a nickname and one of three broad reading levels; it does not collect a birth date. Companion-name screening is a basic local aid using normalized whole-word checks, not guaranteed moderation. A grown-up reviews and can correct the name before entry.

Microphone access is never automatic. Browser speech recognition, where available, is started only after a clear button press; the typed path remains available. A production release must disclose the browser/vendor speech-processing behavior, obtain verified adult consent where required, and provide deletion and retention controls.

Before production use, the project needs independent reviews for child safeguarding, applicable privacy law, early-literacy pedagogy, content rights, accessibility, and platform family policies. AI output must be constrained, moderated, tested across dialects, and reviewable by adults.

The prototype forbids open child profiles, full names, direct messaging, behavioural ads, location, public school identity, tradable child rewards, manipulative engagement, or shame-based feedback.
